A — Cases-centric (recommended)

Each investigation is a first-class Case with its own corpus, method, chain-of-custody, and report. The left rail is your case list (status dots: open / running / done). Inside a case, the work is a 5-step path. Mental model matches how a forensic linguist or lawyer thinks: "open the Doe case." Multiple cases live side-by-side; nothing leaks between them.

B — Stages-centric (today's structure)

Five top-level pages — Ingest, Study, Run, Results, Forensic — operating on whatever's loaded into session state. Familiar to existing users, less work to migrate. But there's no "what am I working on?" anchor: switching projects means re-ingesting, and the Forensic tab feels parallel to the rest rather than a different mode of using the same case.

If A, the existing Ingest/Study/Run/Results/Forensic pages become the 5 steps inside a Case. Nothing is thrown away — only re-homed. The flat bitig run study.yaml CLI still works untouched.
